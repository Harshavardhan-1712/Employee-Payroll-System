Project Structure

Modular Architecture

Five focused Python modules with clear single responsibilities — easy to trace, extend, or replace.

🚀

main.py

Entry point. Drives the interactive menu loop, handles all user input, seeds demo data on first run, and delegates to the appropriate module.

entry point
🗄️

database.py

SQLite CRUD layer. Manages connection lifecycle, table initialisation, and all read/write operations for the employees table.

data layer
💰

payroll.py

Salary calculation engine. Computes overtime pay, gross salary, and net salary per employee. Renders full payroll tables and individual payslips.

business logic
📊

analytics.py

matplotlib-powered dashboard. Generates four styled dark-mode charts and prints KPI summaries to the console. Saves PNGs to /reports/.

visualisation
💾

export.py

Report exporter. Writes timestamped CSV files and styled multi-sheet Excel workbooks (via openpyxl) to /exports/.

Core Logic

Salary Calculation

Three-step formula — transparent, parameterised, and easy to extend with tax brackets or PF deductions.

# ── Overtime ────────────────────────────────────────────── overtime_pay = overtime_hours × ₹200 # configurable rate # ── Gross Salary ────────────────────────────────────────── gross_salary = basic_salary + bonus + overtime_pay # ── Net Salary ──────────────────────────────────────────── net_salary = gross_salary − deductions

Data Model

Database Schema

Single-table SQLite design — zero dependencies on an external server. The DB file is auto-created on first run.

CREATE TABLE employees ( emp_id INTEGER PRIMARY KEY, name TEXT NOT NULL, department TEXT NOT NULL, designation TEXT NOT NULL, basic_salary REAL NOT NULL, bonus REAL DEFAULT 0, -- performance / discretionary deductions REAL DEFAULT 0, -- tax, insurance, etc. attendance INTEGER DEFAULT 0, -- working days present overtime INTEGER DEFAULT 0 -- hours beyond shift );
